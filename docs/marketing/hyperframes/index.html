cc-clip
remote AI coding over SSH

Screenshots should cross SSH too.

Paste local images into remote Claude Code, Codex CLI, and opencode sessions without manual upload detours.
the workflow problem

Local image. Remote agent. One paste.

Your screenshot stays local. The remote Linux session cannot see the Mac clipboard by default.
cc-clip bridges only the needed path. A local daemon and SSH RemoteForward carry image data on demand.
The remote tool keeps its normal paste flow. Shims and bridges serve the clipboard reads the agent already makes.
